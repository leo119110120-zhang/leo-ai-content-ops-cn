# 内容发布检查表

每篇内容必须逐项检查。任何阻塞项未通过，都不进入人工上传。

## 1. 需求与选题

- [ ] 明确谁有这个问题，以及他通常去哪里找解决方案。
- [ ] 至少一个可追溯来源支持选题，不把模型输出当来源。
- [ ] 已区分内容兴趣、咨询意向和真实付款证据。
- [ ] 写清已知事实与未知项，没有把推断写成事实。
- [ ] 选题达到项目门槛，并经过人工选择。

## 2. 来源与事实

- [ ] 所有数字、案例、产品能力和平台表现都能回到来源。
- [ ] 当前性网页记录了检索日期。
- [ ] 没有虚构用户、评论、成交、收益或本人体验。
- [ ] 引用和改写不过度复制原文。
- [ ] 医疗、法律、金融、政策等高风险内容已使用权威最新来源或放弃发布。

## 3. 平台适配

- [ ] 公众号和小红书共享事实，但不是逐字复制。
- [ ] 标题承诺在正文前 30%得到兑现。
- [ ] 小红书只解决一个主要问题，首图可独立表达。
- [ ] 公众号包含背景、证据、方法、边界和可执行步骤。
- [ ] 结尾问题自然，不机械求赞、收藏或关注。

## 4. 图片与隐私

- [ ] 公众号封面和小红书图片尺寸正确、文字无溢出。
- [ ] 中文长文字使用确定性排版，不依赖生图模型直接写字。
- [ ] 手机号、订单号、头像、聊天记录和私人路径已脱敏。
- [ ] AI 示意图、推断图和模拟数据已做必要说明。
- [ ] 图片来源、生成方式和授权边界清楚。

## 5. 人工终审与发布

- [ ] 已在本地审核页通过终稿。
- [ ] 发布包与审核内容一致。
- [ ] 账号、标题、封面、正文、话题和链接均已在平台预览中复核。
- [ ] 人工发布；系统没有保存平台密码、Cookie 或 OTP。
- [ ] 发布后记录内容 ID、平台链接和发布时间。

## 6. 发布后反馈

- [ ] 已安排 24 小时数据记录。
- [ ] 已安排 72 小时数据记录。
- [ ] 已安排 7 天数据记录。
- [ ] 评论、私信、询价和付款分层记录。
- [ ] 不因单篇内容表现立即改变长期定位。
